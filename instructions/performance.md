# 성능 최적화 지침

## 일반 원칙

- 조기 최적화 금지: 먼저 동작하게, 그 다음 최적화
- 측정 후 최적화: 병목 지점을 확인한 후 개선
- 트레이드오프 고려: 가독성 vs 성능

## 프론트엔드 성능

### 렌더링 최적화
- 불필요한 리렌더링 방지
- 메모이제이션 활용 (memo, useMemo, useCallback)
- 가상화(virtualization): 대량 리스트 렌더링 시

### 번들 최적화
- 코드 스플리팅: 필요한 코드만 로드
- 지연 로딩(lazy loading): 화면에 보일 때 로드
- 트리 쉐이킹: 사용하지 않는 코드 제거

### 이미지 최적화
- 적절한 포맷 사용 (WebP, AVIF)
- 반응형 이미지 (srcset)
- 지연 로딩 (loading="lazy")
- 적절한 크기로 리사이징

### 네트워크 최적화
- 요청 수 최소화 (번들링, 스프라이트)
- 캐싱 활용 (브라우저, CDN)
- 압축 (gzip, brotli)

## 백엔드 성능

### 데이터베이스
- 인덱스 적절히 활용
- N+1 문제 방지 (eager loading)
- 쿼리 최적화
- 커넥션 풀링

### API 최적화
- 페이지네이션 필수
- 필요한 필드만 반환
- 응답 캐싱
- 압축 적용

### 비동기 처리
- I/O 작업은 비동기로
- 무거운 작업은 백그라운드 처리
- 적절한 타임아웃 설정

## 측정 도구

- Lighthouse (프론트엔드)
- Chrome DevTools Performance 탭
- 서버 APM 도구
